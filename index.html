Web Servers

A designated homepage is what is referred to as the file in the root directory that is referenced as the default file to return whenever a GET request is initiated for the "/" request on a web-server. The "/" request is typically mapped to the file name "/index.html". The Apache web-server, which OSU Engineering uses for their server software, uses this standard.

The Web Dev/Inspector Network tab provides useful insights into how the web page is accessed and retrieved. The status code upon inspection of the file for both local and web-server access has a value of 200. Codes ranging from 200-299 indicate successful responses, and the code of 200 specifically indicates that the response body has all the necessary information. The request methods between both local and server-side access can be seen as "GET". This indicates that we are requesting to receive the data from the source, be it local or from the web server. The types displayed for both local and server-side files display a content type of "text/html". This indicates that the content received is simple text with HTML language. The URL (Uniform Resource Locator) for the local file is, as expected, different from that of the server-side's URL. The local URL is the path to the local file on the machine. For example, my Macbook machine starts with "/users" and ends with the "/M4/a4-grinne/index.html". The URL from the web-server is much different and instead starts with "web.engr.oregonstate.edu", which is the OSU web server website that hosts a student's site. The ports also differ between local and server-side inspection. On the local side, there is no port because we are not accessing information from the internet. On the server-side, the port is indicated as 443, which is the standard port for web pages that are encrypted with HTTPS. The favicon name is only present when inspecting the web-server file, and represents the logo/icon that this OSU server automatically provides.

The favicon.ico file has a status of 200 because the file is automatically provided within this OSU web-server. The main.css and main.js files have a status code of 404 because although these are referenced in the HTML code, the web-server does not provide these services.

The URL of an internet address contains several parts. We will go through each part of the URL where this file resides, at https://web.engr.oregonstate.edu/~grinne/a4-grinne/. The scheme is the protocol the web client uses to request the resource. For this file, it is HTTPS. The server name (domain) is the name of the machine where the IP address of the file is hosted. In this case it is "oregonstate.edu". The subdomain is the prefix to the domain name and is a subsection of that domain. Here, it is "web.engr". The port on the server is the port on the machine where the data is transmitted. The standard port number for HTTPS is 443, and so it is omitted from the URL. The path to resource is the path from the server to the requested destination/file. In this case, it is "~grinne/a4-grinne/". Note the final "/", which is mapped to "index.html" as mentioned when discussing homepages. The item on the page (or fragment) is not present in this URL. If present, this would indicate the part of the page to locate (or scroll to) and display. Query parameters are also not present within the URL, as we are not requesting specified filtering of additional data.

Frontend Design

Frontend design involves creating an appropriate user interface and experience on the web. It includes designing appropriate visuals on the page, the graphical user-interface (GUI), and the overall interactive experience. Visual design should have a consistent color scheme, font and typography scheme, a photography, icon, and illustration scheme, and navigation systems that make sense.

Effectiveness
Effectiveness is the measure of the degree to which a user can successfully achieve their goals on the website.
Efficiency
Efficiency measures how quickly the user can complete tasks on the website, ideally with the least number of steps.
Ease of Navigation
Intuitability is the measure of how easily new users can intuit the function and navigation of the website.
Error-free
The website should be polished as much as possible to avoid commonplace errors.
Enjoyability/Engagement
Enjoyability and engagement is the overall measure of how well the website works, and the satisfaction of the overall experience.

Page Layout Tags: The <header> tag is used as the banner of the site, and is typically the same across pages/links. The <main> tag holds the primary content of the webpage. The <section> tag defines sections, and can be used to group content together. The <article> tag is usually used within a section, and holds a specific topic within that section. The <footer> tag is placed after the closing main tag and contain legal/contact information, and links to important pages. The <nav> tag defines a set of navigation links to different pages.

  1. External links: Anchors link to an external URL using the 'href' attribute. For example: MDN's Developer Website
  2. Internal links: Anchors link to internal content from one text item to another text item within the same page, by using IDs. For example: Go to the Web Server
  3. Page-to-page links: Anchors link to other pages within the same website using a relative URL. For example: Contact

Optimizing Images

File Name
The file name should be descriptive so that search engine optimization (SEO) can be utilized to identify the file across the web.
File Size
File sizes should be made as small as possible so that loading times are quick and seamless. Proprietary software usually has varioius techniques to resize images appropriately.
Exact Dimensions
Reducing and cropping the size of images to alignw witht the spacing of the webpages is good practice.
File Format
Images can natively be several formats. On the web, JPG is the standard photo format for best processing. Line-art images are usually GIF, PNG, or SVG, and other specialized graphics should be PNG.
Resolution
Monitors have varying levels of pixels per inch (ppi) that they can render, typically from 72 to 300+ ppi. As 72 ppi is the default, it is recommended to provide images of at least that size.
Color Mode
RGB color mode is used with PNG, JPG, SVG, and WebP file types, while Indexed is used for GIF.

Favicons

Favicons can be thought of as logos that represent the company, app, or general brand being displayed. They can be GIF, PNG, ICO, or SVG file formats. The favicon symbol image is tested as different file formats so that it works on multiple browsers/devices. Then, when the browser has access to the symbol, it will use that symbol in places such as a tab, app logo, search engine results, bookmarks, etc. Modern techniques to accomodate several devices have a higher quality favicon and a lower quality one, so that the appropriate image can be used in places that will allow each. For example, a high quality favicon should be used with an app logo, while a lower quality favicon could be present in the browser tab.
